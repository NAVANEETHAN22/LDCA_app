1. App Overview & Purpose

LDCA (Lightweight Data Classification App) is a fully self-contained Android application that lets users perform binary classification using standard machine-learning models — entirely on-device, without any internet connection or cloud server.

Users can load a CSV dataset (from local storage or a URL), choose a classification model (or let the app choose automatically), run the full ML pipeline, view rich result visualizations, and download a PDF analysis report — all from their Android phone.

Key Capabilities

  • Load CSV datasets from device storage, HTTP/S URL, or manually enter a single data point
  • Run 4 different classification algorithms
  • Intelligent Auto Model Selection based on dataset shape
  • Real-time training progress updates on screen
  • View Accuracy, Precision, Recall, F1, Confusion Matrix, ROC Curve, PR Curve, Feature Importance, and 5-Fold Cross Validation
  • Export a complete PDF report to the Downloads folder

3. Screens & User Flows

Screen 1 — MainActivity (Home Screen)

The entry point of the app. Displays 3 buttons that let the user choose how to provide data:

  • Upload CSV → opens UploadCSVActivity
  • URL CSV → opens URLCSVActivity
  • Manual Input → opens PayloadActivity

Screen 2 — UploadCSVActivity (Upload CSV from Device)

Lets the user pick any .csv file from their device storage using the Android file picker.

Step-by-step what happens:

  1. User taps Choose File → system file picker opens (filters for text/*)
  2. CSV is read in a background thread with a real-time percentage progress bar (reads file size from ContentResolver to calculate %)
  3. The first 10 rows are displayed in a scrollable preview table
  4. Dataset info (row count, column count) is shown
  5. User selects a model from the Spinner (dropdown): Auto Select, Naive Bayes, Logistic Regression, Decision Tree, Random Forest
  6. An ℹ info button opens a dialog explaining all models and auto-select criteria
  7. User taps Run Analysis → ML pipeline runs in background thread
  8. Spinner is locked during analysis (shows Toast if touched)
  9. After training + prediction, app navigates to ResultActivity
Max rows limit: The dataset builder supports unlimited rows from the CSV; the raw list is fully loaded up to 5000 rows per the MAX_ROWS constant in UploadCSVActivity.

Screen 3 — URLCSVActivity (Load CSV from Internet)

Accepts an HTTP/HTTPS URL pointing to a raw CSV file.

How it works:

  1. User pastes a URL (e.g., a raw GitHub CSV link)
  2. Taps Load → HttpURLConnection downloads the CSV in a background thread (10s connect timeout, 15s read timeout)
  3. Status text shows row count × column count once loaded
  4. Same model spinner + info button as the Upload screen
  5. Taps Run Analysis → same ML pipeline → ResultActivity

Screen 4 — PayloadActivity (Single Data Point Prediction)

Allows the user to manually type feature values and get a one-off classification prediction.

How it works:

  1. 5 numeric input fields are dynamically generated (labeled Feature 1 – Feature 5)
  2. User types decimal values into each field
  3. Selects a model from the spinner
  4. Taps Predict → runs the chosen model on a single-row dummy dataset
  5. Result is displayed as: "Predicted Class (ModelName): 0 or 1"
  6. A Download PDF button appears → saves a minimal PDF report to Downloads
Note: Since there is only one data row, the model trains on that single row and predicts it — this is a simple proof-of-concept for ad-hoc single-observation classification.

Screen 5 — ResultActivity (Analysis Dashboard)

The richest screen in the app. Receives all result data via Android Intent extras and renders a full dashboard:

4. Machine Learning Models

All four models are implemented from scratch in pure Java — no external ML libraries are used.

4.1 Naive Bayes

ProbabilisticSmall Datasets

Implements Gaussian Naive Bayes. Assumes feature values follow a Gaussian (normal) distribution within each class.

Training:

  • Counts samples per class to compute prior probabilities: P(class)
  • Computes per-class, per-feature mean and variance

Prediction:

  • For each class, computes log-posterior: log P(class) + Σ log P(xj | class)
  • Uses Gaussian PDF: score -= (x-mean)² / (2*var + 1e-9)
  • Returns the class with the highest score

Feature Importance:

Uniform — each feature gets 1 / numFeatures importance (NB treats all features equally).

4.2 Logistic Regression

Linear BoundaryMedium Datasets

Implements binary logistic regression with Gradient Descent.

Hyperparameters:

  • Learning rate lr = 0.01
  • Epochs: 500

Training:

  • For each epoch, iterates over every sample
  • Computes sigmoid output: σ(w·x)
  • Error = y - σ(w·x)
  • Weight update: w += lr × error × x

Prediction:

Returns 1 if σ(w·x) > 0.5, else 0. The raw sigmoid output is used as the probability score for ROC/PR curves.

Feature Importance:

Absolute weight values normalized by the max weight: |wj| / max(|w|)

4.3 Decision Tree

Single Split StumpHigh Dimensional

Implements a Decision Stump (depth-1 decision tree). Finds the single best feature split.

Training:

  • Iterates over all features and all unique split thresholds
  • For each split, counts correct predictions
  • Selects the feature + threshold pair that gives highest accuracy
  • Stores: featureIndex, threshold, leftLabel, rightLabel

Prediction:

Returns leftLabel if x[featureIndex] <= threshold, else rightLabel.

Feature Importance:

Only the selected split feature gets importance = 1.0; all others = 0.

4.4 Random Forest

EnsembleLarge Datasets

An ensemble of 10 Decision Stumps, each trained on a bootstrap sample.

Training:

  • Fixed random seed 42 for reproducibility
  • For each of the 10 trees:
    • Creates a bootstrap sample (sample with replacement, same size as original)
    • Trains a DecisionTree on the bootstrap sample
  • Feature importances accumulate by counting which feature each tree split on
  • Importances are normalized: count[j] / nTrees

Prediction (Majority Vote):

Each of 10 trees votes; the majority class wins.

Probability Score (for ROC/PR):

Fraction of trees that voted class 1: votes1 / nTrees

5. Auto Model Selection Logic

When the user selects "Auto Select" from the model spinner, the AutoModelSelector class examines the dataset shape and picks the best model using these rules (evaluated in order):

ConditionModel ChosenReason
rows < 200Naive BayesSmall datasets — probabilistic model works well with limited samples
cols > 10Decision TreeHigh-dimensional data — stump handles feature splits naturally
rows ≥ 500Random ForestEnough data for ensemble — provides better accuracy & robustness
OtherwiseLogistic RegressionMedium size, few features — linear decision boundary is reliable

A human-readable reason string is shown as a banner on the Result screen (e.g., "🤖 Auto-Selected: Random Forest — dataset has 500+ rows..."). The same info dialog is accessible via the ℹ info button on the analysis screens.

Step 1 – Load CSV

Background thread reads the CSV line-by-line into a List<String[]>. Progress (%) is reported to the UI thread every 2%.

Step 2 – Parse Rows

Each line is split on ",". The first row is treated as the header (column names). Feature names are extracted from header (all columns except the last).

Step 3 – Build DataSet (DatasetBuilder)

  • LabelDetector identifies the label column (currently assumed to be the last column)
  • Numeric cells are parsed as double
  • String/categorical cells are encoded as integers using CategoricalEncoder (maps unique strings to 0, 1, 2, …)
  • Label column values are mapped to integers via a separate labelMap
  • Result: a DataSet object with double[][] X (feature matrix) and int[] y (labels)

Step 4 – Select Model

If "Auto Select" → AutoModelSelector.chooseModel(dataset) and getReason(dataset) are called. Otherwise, the user's spinner choice is used.

Step 5 – Train

The selected model's .train(dataset) method is called. Training status ("Training RandomForest...") is displayed on the UI thread.

Step 6 – Predict All

Every row in the dataset is predicted. Progress is reported every 2000 rows. Two arrays are built: int[] predictions and double[] scores (probability for class 1).

Step 7 – Evaluate (ResultEngine)

Calls ResultEngine.evaluate() which computes: accuracy, confusion matrix, precision, recall, F1, ROC curve, PR curve, AUC, and 5-fold cross-validation (if a DataSet is passed).

Step 8 – Show Results

All arrays and scores are bundled into an Android Intent and ResultActivity is started.

7. Evaluation Metrics

Accuracy

Fraction of correctly classified samples.

correct / total

Precision

Of all predicted positives, how many were truly positive?

TP / (TP + FP)

Recall (Sensitivity)

Of all actual positives, how many did the model find?

TP / (TP + FN)

F1 Score

Harmonic mean of Precision and Recall. Balances both.

2 × P × R / (P + R)

AUC-ROC

Area Under the ROC Curve. 1.0 = perfect, 0.5 = random.

Trapezoidal rule on FPR/TPR

CV Mean Accuracy

Average accuracy across 5 stratified folds.

mean(fold1..fold5)

Confusion Matrix

Predicted 0Predicted 1
Actual 0TN (True Negative)FP (False Positive)
Actual 1FN (False Negative)TP (True Positive)
A small epsilon 1e-9 is added to all denominators to avoid division-by-zero in Precision, Recall, and F1 calculations.

ROC Curve Computation

Thresholds are determined by sorting all samples by descending score. For each threshold (each sample removed one at a time), FPR and TPR are computed. The resulting (FPR, TPR) pairs form the ROC curve. AUC is computed using the trapezoidal rule.

5-Fold Cross Validation

The dataset is partitioned into 5 equal folds. For each fold, the model trains on the remaining 4 folds and validates on that fold. The mean and per-fold accuracies are returned via CrossValidation.CVResult.

10. Utility Classes

DataSet.java

Simple data container: double[][] X (feature matrix), int[] y (labels), int rows, int cols.

DatasetBuilder.java

Converts List<String[]> (raw CSV rows) into a DataSet. Handles both numeric values (parsed as double) and categorical strings (encoded via CategoricalEncoder). Delegates label column detection to LabelDetector.

LabelDetector.java

Determines which column is the target label. Currently assumes the last column is the label column.

CategoricalEncoder.java

Ordinal-encodes categorical string values to integers. Maintains a HashMap<String, Integer> per column. Each new unique value gets the next integer ID (0, 1, 2, …).

ResultBundle.java

A plain data-holding class (like a struct) that carries all computed results between ResultEngine and the UI layer: accuracy, precision, recall, f1, confusion matrix, CV results, ROC/PR arrays, feature importances.

ResultEngine.java

Orchestrates metric computation. Has two overloads: a simple one (accuracy + confusion matrix only) and a full one that also runs cross-validation, ROC/PR curves, and feature importance.
